Politique de confidentialité

FocusGate / 守界, appelé ci-dessous « FocusGate », est une extension de navigateur qui limite les sites distrayants selon les groupes de règles configurés par l’utilisateur. Elle aide à protéger les limites d’attention pendant le sommeil, le travail, les études et les périodes de recul numérique.

La confidentialité fait partie de la limite du produit. Le MVP actuel fonctionne localement par défaut. Il ne propose pas de compte, n’effectue pas de synchronisation cloud et ne vend pas les données utilisateur.

Données que nous traitons

FocusGate stocke les données suivantes localement dans votre navigateur afin de fournir les fonctions de limite de sites web que vous configurez :

  • Paramètres des groupes de règles : nom du groupe, état activé, horaires, jours actifs, domaines restreints, texte d’engagement, délai de rappel, intensité de blocage, paramètres de déverrouillage temporaire et configuration d’affichage de la page de blocage.
  • Règles de sites web : domaines que vous ajoutez aux groupes de règles, par exemple youtube.com, bilibili.com ou d’autres sites choisis par vous.
  • État de déverrouillage temporaire : domaine déverrouillé, groupe de règles associé, heure de début, heure d’expiration, durée, mode de blocage et motif optionnel.
  • Événements statistiques locaux : blocage, déverrouillage, ajout de site, pause, rappel, effacement et événements similaires, avec le domaine, le groupe de règles et l’horodatage associés.
  • Contenu d’affichage personnalisé : HTML statique de page de blocage, HTML de page de transition, titre, description et configuration du bouton principal saisis dans les paramètres.

Données que nous ne collectons pas

La version actuelle ne collecte pas activement, ne téléverse pas et ne vend pas les informations suivantes :

  • Contenu textuel des pages.
  • Contenu des champs de formulaire.
  • Comptes, mots de passe ou identifiants.
  • Titres des pages.
  • Historique complet de navigation.
  • Contenu des pages de navigateur non HTTP(S).
  • Informations personnelles sans rapport avec les fonctions de FocusGate.

FocusGate doit seulement déterminer si le domaine de l’URL actuelle correspond aux groupes de règles que vous avez configurés. Les événements statistiques sont par défaut au niveau du domaine, pas au niveau du contenu des pages.

Stockage des données

Les données de la version actuelle sont stockées localement par défaut, notamment dans chrome.storage.local pour l’extension Chrome et dans un repli de stockage local utilisé en environnement de développement.

Ces données restent principalement sur votre appareil et servent à déterminer si un site doit être limité, afficher les pages de blocage et les rappels, exécuter les déverrouillages temporaires et leur expiration automatique, ainsi qu’afficher les statistiques locales dans les paramètres.

Le MVP actuel ne propose pas de connexion à un compte, de synchronisation cloud ni de service d’analyse distant.

Préférence de langue du site public

Ce site public lit les valeurs navigator.languages ou navigator.language du navigateur pour choisir les pages en chinois, anglais ou français depuis l’entrée chinoise par défaut. Le sélecteur de langue peut enregistrer un code de langue dans le localStorage de votre navigateur afin de mémoriser votre choix lors d’une prochaine ouverture de l’entrée par défaut.

Cette préférence sert uniquement au routage linguistique du site public. Ce site statique ne l’envoie pas au serveur du développeur et elle ne modifie pas la façon dont l’extension traite les données.

Partage des données

La version actuelle ne téléverse pas vos groupes de règles, domaines restreints, motifs de déverrouillage, événements statistiques ou HTML personnalisé vers les serveurs du développeur, et ne les vend pas à des annonceurs, courtiers en données ou autres tiers.

Si une version future ajoute la synchronisation cloud, les comptes, l’analyse de crash ou des fonctions multi-appareils, cette politique de confidentialité devra être mise à jour au préalable, et le produit devra expliquer clairement les nouveaux usages de données et les choix proposés aux utilisateurs.

L’utilisation des données utilisateur par FocusGate respectera la Chrome Web Store User Data Policy, y compris les exigences Limited Use. FocusGate n’utilisera les données que pour fournir ou améliorer les fonctions de limite de sites web clairement décrites par l’extension.

Utilisation des autorisations

  • storage : enregistre les groupes de règles, sites, états de déverrouillage et statistiques locales.
  • tabs : lit ou met à jour les onglets du navigateur afin de rediriger les sites restreints vers la page de blocage quand une règle correspond.
  • activeTab : permet à la fenêtre de l’extension d’identifier le domaine de l’onglet actuel et d’ajouter le site actuel à un groupe de règles.
  • webNavigation : observe la navigation de premier niveau pour déterminer si le site visité doit être bloqué.
  • alarms : planifie les rappels, les vérifications de blocage et le nettoyage des déverrouillages expirés.
  • notifications : affiche des notifications système peu avant le début d’une règle.
  • Autorisation d’hôte <all_urls> : vérifie les domaines sur tout site HTTP(S) configurable par l’utilisateur et affiche le rappel anticipé.

FocusGate n’utilise pas ces autorisations pour lire le texte des pages, les champs de formulaire, les informations de compte ou les titres de page.

Contrôle utilisateur

Vous pouvez utiliser la page de paramètres de l’extension pour créer, modifier, activer ou supprimer des groupes de règles, ajouter ou retirer des domaines restreints, modifier les rappels, l’intensité de blocage et les règles de déverrouillage temporaire, effacer les statistiques locales, ou désactiver ou désinstaller l’extension.

La suppression des données après désinstallation dépend du fonctionnement du stockage des extensions Chrome / Edge. Vous pouvez aussi effacer les statistiques locales depuis la page de paramètres avant de désinstaller.

Confidentialité des enfants

FocusGate n’est pas conçu spécifiquement pour les enfants. La version actuelle ne propose pas de compte et ne demande pas activement aux utilisateurs de fournir leur âge, leur nom, leurs coordonnées ou d’autres informations d’identification.

Modifications de cette politique

Si des versions futures modifient substantiellement le traitement des données, par exemple en ajoutant des comptes, une synchronisation cloud, une sauvegarde distante, de la télémétrie ou des services tiers, nous mettrons à jour cette politique de confidentialité et expliquerons les changements à un endroit approprié.
