# Store Documentation Guidance

This directory is the source of truth for Chrome Web Store listing visuals, copy direction, and capture plans.

- Keep agent-facing instructions in English.
- Keep store-facing copy, screenshot captions, and visual text in Chinese unless a task explicitly targets another locale.
- Treat `prd.md`, `DESIGN.md`, and the live React entry points as upstream sources. Reference them instead of duplicating product scope.
- Do not describe unbuilt features as available store capabilities.
- When Chrome Web Store requirements change, update the plan with a dated note and link to the authoritative Chrome for Developers page.
- Mark generated mockups and SVG templates clearly as placeholders. Final screenshots must come from the built extension UI.
- Record capture assumptions, viewport sizes, seed data, and any remaining manual work so a fresh agent can reproduce the asset set.
