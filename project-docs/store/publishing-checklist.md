## Pre-Submission Checks

Run these from the repository root:

```bash
npm run typecheck
npm test
npm run test:e2e
npm run build
npm run store:assets
npm run store:screenshots
npm run package:store
```

Then manually inspect:

- `dist/manifest.json` exists and uses `"manifest_version": 3`.
- `dist/` can be loaded as an unpacked extension in Chrome or Edge.
- The packaged zip contains files at the zip root, not inside a nested `dist/` folder.
- The packaged zip does not include `store/` source assets, screenshots, or source maps.
- No executable remote JavaScript or WASM is loaded from the network.
- The extension still has a single purpose: user-configured website boundaries for sleep, focus, study, and digital-withdrawal scenarios.

## Dashboard Fields

Use the Chrome Web Store Developer Dashboard tabs in this order.

### Package

Upload the zip generated by:

```bash
npm run package:store
```

The output path is printed by the script and should look like:

```text
artifacts/chrome-web-store/focusgate-<version>.zip
```

### Store Listing

- Name: use the final decision in `project-docs/store/submission-metadata.md`.
- Summary: use `project-docs/store/store-listing-draft.md`.
- Detailed description: use `project-docs/store/store-listing-draft.md`.
- Category: `Productivity`.
- Language: `zh-CN` for the first Chinese listing.
- Website/support URLs: use `project-docs/store/submission-metadata.md`.
- Screenshots: upload final reviewed screenshots from `public/store/screenshot-*.png`.
- Small promotional tile: upload `public/store/promo-tile-440x280.png`.
- Marquee image: upload `public/store/marquee-1400x560.png` if the dashboard accepts it for the item.

### Privacy

Single purpose:

```text
FocusGate / 守界 lets users create local rule groups that limit access to user-selected websites during scheduled focus, sleep, study, or digital-withdrawal windows.
```

Privacy policy URL:

```text
https://aigotowork.github.io/focusgate/privacy/
```

Data handling summary:

- The extension processes URL/domain information only to evaluate user-configured rules, show reminders or block pages, and generate local domain-level stats.
- The extension does not read page body text, form inputs, account credentials, or page titles.
- The current MVP does not upload rule groups, restricted domains, unlock reasons, or stats to a developer server.
- The current MVP does not sell data, use data for advertising, or provide remote analytics.

Sensitive permission notes are in `project-docs/store/permission-justification-draft.md`.

### Distribution

MVP decision:

- Pricing: `Free`.
- Visibility: use `Trusted testers` for a final private smoke test, then switch to public launch when accepted.
- Regions: all available regions.
- API note: trusted testers is configured in the Dashboard visibility settings. Chrome Web Store API v2 submission still uses `DEFAULT_PUBLISH` or `STAGED_PUBLISH`; it does not use a `TRUSTED_TESTERS` publish type.

### Test Instructions

Paste a concise reviewer flow:

```text
Install the extension and open the welcome page. Open Options, keep the default "晚安守护" rule group, or create a new "工作时间专注" rule group. Add a test domain such as example.com, set the schedule to the current time window, and save. Visit the domain in a normal tab; the extension should redirect the top-level navigation to its block page and show the matching rule group, blocked domain, commitment, primary action, and temporary unlock path. To test reminders, set a rule to start within the configured reminder window, save, and visit the listed domain before the start time; the page should show a quiet reminder overlay. All settings and stats are stored locally.
```

## Known Review Risks

- `<all_urls>` is broad. The justification must stay precise: users can add arbitrary domains, and host access is used for URL/domain evaluation plus the reminder overlay.
- The content script runs on HTTP(S) pages. Keep it limited to the low-stimulation reminder overlay and do not read page contents.
- MV3 review checks compiled output. Search `dist/` for remote executable code before upload when new dependencies are added.
- Listing copy must not claim unbuilt features such as cloud sync, account login, mobile app blocking, uninstall prevention, or AI recommendations.
- Store images must show real UI for screenshots. Promotional images can be branded graphics, but should not include misleading badges or status claims.
