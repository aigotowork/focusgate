# FocusGate / 守界 产品需求文档

> 全球品牌：**FocusGate / 守界**
> 标语：**该专注时，守住你的注意力边界**
> 产品形态：**Manifest V3 Chrome / Edge 浏览器扩展**
> 当前定位：**用多个规则组，为睡眠、工作/学习专注、数字戒断等场景建立可执行的网站访问边界。**
> 默认规则组：**晚安守护**，仅代表睡眠场景，不再作为全局产品名。

---

## 1. 品牌决策

### 1.1 全局品牌

产品全局品牌统一为 **FocusGate / 守界**。

- **FocusGate**：面向国际化、商店上架、技术文档和品牌资产。
- **守界**：面向中文用户，表达“守住边界，而不是惩罚自己”。
- **标语**：该专注时，守住你的注意力边界。

### 1.2 命名边界

| 名称 | 使用范围 | 说明 |
|---|---|---|
| FocusGate / 守界 | 产品品牌、扩展名称、全局导航、商店描述 | 代表通用注意力边界工具 |
| 晚安守护 | 默认睡眠规则组 | 只用于睡眠场景模板 |
| 工作时间专注、备考冲刺、周末戒断等 | 用户自定义规则组 | 由规则组承担具体场景语义 |
| 专注边界 / Focus Boundary | 历史名称 | 不作为新 PRD 的品牌主轴 |

### 1.3 文案原则

中文用户界面应温和、清晰、低刺激。产品不是“强制管控工具”，而是帮助用户在清醒时预设边界，并在容易分心时执行边界。

---

## 2. 产品定位

### 2.1 一句话定位

**FocusGate / 守界 是一个本地优先的浏览器扩展，帮助用户按规则组阻断容易夺走注意力的网站。**

### 2.2 核心问题

用户在不同场景下都可能被高刺激网页带走：

- 睡前本想休息，却继续刷视频、社区、小说和新闻；
- 工作或学习时从查资料滑向娱乐网站、社交网站或购物网站；
- 数字戒断期间反复打开短内容、论坛、直播和信息流；
- 明知道应停止，但临时破例太容易，缺少可见的边界和复盘反馈。

### 2.3 核心价值

FocusGate 不追求无法绕过的绝对强制，而追求四件事：

1. 在正确时间识别当前网站是否越界；
2. 用规则组表达不同生活场景的边界；
3. 通过阻断页、提醒、解锁摩擦和承接页帮助用户转身离开；
4. 默认本地保存设置和统计，让用户能复盘而不牺牲隐私。

## 3. MVP 范围

### 3.1 必须包含

| 模块 | MVP 要求 |
|---|---|
| 规则组 | 支持多个规则组；每组独立启用、命名、排程、站点、承诺语、提醒、阻断模式、解锁策略和阻断页配置 |
| 默认模板 | 首次默认创建“晚安守护”，用于睡眠场景 |
| 时间排程 | 支持开始时间、结束时间、生效星期、跨天区间和提前提醒窗口 |
| 网站规则 | 支持按域名添加、删除和匹配站点；主域名应匹配子域名且避免误伤相似域名 |
| 阻断决策 | 访问命中规则组的网站时，按排程、暂停、解锁状态和阻断强度做判断 |
| 冲突处理 | 多个规则组同时命中时，严格程度高者优先：`strict > standard > gentle` |
| 阻断页 | 展示命中的规则组、站点、承诺语、阻断页配置、主操作和临时解锁入口 |
| 自定义静态 HTML | 支持每组自定义阻断页静态 HTML，并以沙盒 iframe 渲染 |
| 承接页 | 支持阻断页主操作打开静态承接页，用于工作清单、睡前收束等下一步内容 |
| 临时解锁 | 解锁仅作用于当前规则组和当前域名；支持时长、次数限制、原因记录和过期恢复 |
| 页面内提醒 | 在提醒窗口内，对即将被阻断的站点展示内容脚本提醒浮层 |
| Popup | 展示当前页面与规则组的关系、今日统计、当前/即将生效状态，并支持把当前网站加入选中规则组 |
| Options | 管理规则组、站点、排程、阻断页、主操作、解锁策略和统计 |
| 统计 | 本地记录阻断、解锁、添加站点、暂停、提醒和清空等域名级事件，展示今日、本周期、近 7 天和高频域名 |
| 本地优先存储 | 设置、解锁会话和事件默认保存在浏览器本地 |

### 3.2 明确不做

| 非目标 | 说明 |
|---|---|
| 手机 App 管控 | MVP 仅覆盖安装扩展的桌面浏览器 |
| 本地应用管控 | 不阻断游戏、播放器、聊天客户端等非浏览器应用 |
| 防卸载、防换浏览器 | 不承诺不可绕过 |
| 云同步和账号系统 | MVP 不上传用户数据，不提供登录 |
| 页面内容分析 | 不读取正文、表单输入、账号信息或页面标题作为统计依据 |
| AI 个性化建议 | 可作为后续方向，不进入当前 MVP |

## 4. 用户场景

### 4.1 睡眠：默认“晚安守护”

用户设置每天 `23:00-07:00` 阻断 `youtube.com`、`bilibili.com`、`reddit.com`。22:30 访问这些站点时出现低刺激提醒；23:00 后再次访问会进入阻断页。用户可以关闭页面、打开睡前承接页，或按规则临时解锁 10 分钟。

### 4.2 工作/学习：工作时间专注

用户创建“工作时间专注”规则组，设定工作日 `09:00-18:00` 限制视频、社交和购物站点。工作中打开 B 站时，阻断页展示“现在是专注时间”，主按钮可跳转到任务清单或打开承接页。

### 4.3 数字戒断：周末减少高刺激内容

用户创建“周末戒断”规则组，周六周日限制短视频、论坛和新闻站。规则组使用严格模式，减少解锁次数，阻断页文案强调“把周末还给现实生活”。

### 4.4 当前页面快速加入

用户正在浏览一个新娱乐网站，点击扩展 Popup，看到当前网站未命中任何规则组，可直接加入“晚安守护”或其他选中的规则组。

### 4.5 解锁后自动恢复

用户因紧急事项临时解锁某域名。解锁只对当前规则组生效；到期后，如果该站点仍在受限时间内，再次访问应重新阻断。

## 5. 核心数据模型

实现来源以 `src/shared/types.ts` 为准，PRD 描述业务含义。

### 5.1 RuleGroup

规则组是产品的核心单位。每个规则组代表一个边界场景。

| 字段 | 含义 |
|---|---|
| `id` | 规则组唯一标识 |
| `name` | 用户可见名称，例如“晚安守护”“工作时间专注” |
| `enabled` | 是否启用该规则组 |
| `schedule` | 生效排程：开始时间、结束时间、生效星期 |
| `sites` | 受限域名列表 |
| `commitment` | 用户给自己的承诺语 |
| `reminderMinutes` | 规则开始前多少分钟提醒 |
| `blockMode` | 阻断强度：`gentle`、`standard`、`strict` |
| `unlockMinutes` | 单次临时解锁时长 |
| `maxUnlocksPerSession` | 每个规则周期允许的解锁次数 |
| `recordUnlockReason` | 是否记录解锁原因 |
| `blockPage` | 阻断页标题、描述、主操作、语气和自定义 HTML |

### 5.2 Schedule

排程支持跨天。例如 `23:00-07:00` 属于一个连续规则周期；凌晨 `01:00` 仍归入前一晚开始的周期。统计和解锁次数应按规则周期归属，而不是简单按自然日切分。

### 5.3 SiteRule

站点规则以域名为核心。用户添加 `youtube.com` 时，应匹配：

- `youtube.com`
- `www.youtube.com`
- `m.youtube.com`
- `music.youtube.com`

不应匹配：

- `fakeyoutube.com`
- `youtube.com.fake-site.com`

### 5.4 UnlockSession

临时解锁必须同时绑定：

- 规则组；
- 域名；
- 解锁开始和过期时间；
- 解锁时长；
- 阻断模式；
- 可选原因。

解锁一个规则组中的 `bilibili.com` 不应放行另一个同时命中的严格规则组。

### 5.5 GuardEvent 与 Stats

事件默认只记录域名级信息，用于统计和复盘。MVP 事件包括：

- `blocked`
- `unlocked`
- `site_added`
- `paused`
- `reminded`
- `cleared`

统计需要支持今日阻断、今日解锁、本周期阻断、本周期解锁、近 7 天趋势和高频阻断域名。

## 6. 页面职责

### 6.1 Popup

Popup 是轻量状态面板，不承担完整配置。

必须展示：

- FocusGate / 守界 品牌；
- 当前标签页域名；
- 当前页面状态：已阻断、已解锁、即将受限、未命中、已暂停、非 HTTP 页面等；
- 命中的规则组或当前选中规则组；
- 今日阻断和今日解锁；
- 当前可加入规则组时，提供“加入到某规则组”的操作；
- 进入设置页入口。

### 6.2 Options

Options 是主要配置中心。

必须支持：

- 创建、选择、重命名、启用和删除规则组；
- 编辑规则组排程、站点、承诺语、提醒时间；
- 选择阻断强度；
- 设置解锁时长、次数限制和是否记录原因；
- 配置阻断页标题、描述、语气、主操作；
- 配置主操作为关闭页面、跳转安全外部 URL 或打开承接页；
- 编辑自定义静态阻断 HTML 和承接页 HTML；
- 预览阻断页和承接页；
- 查看统计、清空本地事件。

### 6.3 Block Page

Block Page 是最强的边界表达页面。

必须展示：

- 被阻断域名；
- 命中的规则组；
- 当前规则组承诺语；
- 规则组阻断页标题和说明；
- 主操作按钮；
- 临时解锁入口；
- 修改设置入口。

要求：

- 临时解锁操作必须留在 React 控制区，不得放入用户自定义 HTML；
- 自定义阻断 HTML 只能作为静态展示内容；
- 解锁成功后才允许返回原网站；
- 阻断页不得循环阻断扩展自身页面。

### 6.4 Handoff Page

Handoff Page 是阻断后的“下一步承接页”，不是解锁通道。

必须支持：

- 根据规则组展示主操作配置的静态 HTML；
- 使用与自定义阻断页一致的沙盒策略；
- 不读取原页面内容；
- 不授予访问目标网站的权限；
- 不修改解锁状态。

### 6.5 Content Reminder Overlay

内容脚本只负责页面内提醒。

必须满足：

- 仅在 HTTP(S) 页面运行；
- 仅当当前域名属于某个即将生效的规则组时显示；
- 提醒结束或条件不满足时移除；
- 使用隔离样式，避免污染网页；
- 不记录页面正文、标题、输入内容。

### 6.6 Background Service Worker

后台负责运行边界规则。

必须支持：

- 初始化默认设置；
- 首次安装打开配置入口；
- 监听顶层导航并评估访问决策；
- 将被阻断页面重定向到 `block.html`；
- 使用 `chrome.alarms` 安排下一次提醒或阻断检查；
- 到点扫描已打开的 HTTP(S) 标签页；
- 发送每个规则周期最多一次的提醒通知；
- 清理过期解锁；
- 记录域名级事件。

## 7. 阻断与提醒规则

### 7.1 访问决策

访问某 URL 时，按以下顺序判断：

1. 扩展页面或非 HTTP(S) 页面：允许；
2. 全局暂停中：允许；
3. 没有启用规则组：允许；
4. 当前域名未命中任何规则组：允许；
5. 命中的规则组不在生效时间：允许；
6. 命中的规则组已有当前域名有效解锁：允许；
7. 多个规则组同时命中时选择最严格者；
8. 其余情况：阻断并记录事件。

### 7.2 提醒窗口

每个规则组可配置 `reminderMinutes`。提醒窗口为：

```text
规则开始时间 - reminderMinutes 到 规则开始时间
```

提醒窗口内：

- Popup 应显示即将受限状态；
- 内容脚本可在命中网站展示提醒浮层；
- Background 可发送每个规则周期一次的浏览器通知；
- 默认不提前阻断，除非后续明确新增预阻断能力。

### 7.3 阻断模式

| 模式 | 目标 | 解锁策略 |
|---|---|---|
| `gentle` | 降低进入门槛，适合新用户 | 最少摩擦，可配置为不限制次数 |
| `standard` | 默认平衡模式 | 有次数限制、原因记录和明确确认 |
| `strict` | 更强边界 | 更短解锁时长、更少解锁次数、更高确认成本 |

实现应以规则组字段为准，不把模式写死为睡眠语义。

## 8. 隐私与安全

### 8.1 隐私原则

| 原则 | 要求 |
|---|---|
| 本地优先 | 设置、解锁和事件默认保存在 `chrome.storage.local` 或开发环境本地存储 |
| 最小记录 | 默认记录域名和事件类型，不记录完整 URL、页面标题、正文或输入内容 |
| 无账号依赖 | MVP 不需要登录，不上传浏览历史 |
| 用户可清理 | Options 应提供清空本地事件或数据的入口 |
| 权限透明 | 扩展权限应能解释为阻断、提醒、存储和当前页面判断所需 |

### 8.2 自定义 HTML 安全

自定义阻断 HTML 和承接 HTML 属于高级静态展示能力。

必须满足：

- 存储大小受限；
- 模板变量必须转义；
- 通过空 `sandbox` iframe 渲染；
- 不暴露扩展 API；
- 不允许用户 HTML 控制解锁、跳转、事件记录或原站访问；
- 外部主操作 URL 必须限制为安全的绝对 `http(s)` URL。

### 8.3 技术约束

Manifest V3 Service Worker 不能依赖长期常驻内存。提醒、阻断检查和过期清理应依赖持久化存储、导航事件和 `chrome.alarms`。

## 9. 验收标准

### 9.1 品牌与文案

| 标准 | 验收方式 |
|---|---|
| 全局品牌使用 FocusGate / 守界 | 检查 PRD、UI 文案和商店文案规划 |
| 标语为“该专注时，守住你的注意力边界” | 检查品牌相关文案 |
| “晚安守护”只作为默认睡眠规则组 | 检查产品描述、页面标题和默认数据 |
| 不再把产品定义为睡眠单一工具 | 检查 PRD 和主要用户流程 |

### 9.2 规则组能力

| 标准 | 验收方式 |
|---|---|
| 可保留默认“晚安守护”并新增工作/学习规则组 | Options 手动验证，必要时用 Playwright 验证 |
| 每个规则组独立保存排程、站点、承诺语、提醒、阻断模式、解锁策略和阻断页配置 | 单元测试和数据模型审查 |
| 跨天排程正确归属规则周期 | `src/shared/time.ts` 单元测试 |
| 域名匹配正确且不误伤相似域名 | `src/shared/sites.ts` 单元测试 |
| 多规则组同时命中时严格模式优先 | 单元测试 |
| 解锁只对当前规则组和当前域名生效 | 单元测试和阻断页手动验证 |

### 9.3 页面闭环

| 标准 | 验收方式 |
|---|---|
| Popup 能识别当前页面状态并支持加入规则组 | Playwright 或手动验证 |
| Options 能完整编辑规则组和阻断页配置 | Playwright 或手动验证 |
| Block Page 显示命中规则组并提供主操作和临时解锁 | Playwright 或手动验证 |
| Handoff Page 只展示承接内容，不授予解锁 | Playwright 或代码审查 |
| 内容提醒浮层只在提醒窗口内出现 | 单元测试和页面验证 |

### 9.4 隐私与安全

| 标准 | 验收方式 |
|---|---|
| 默认不上传数据 | 代码审查 |
| 默认不记录正文、输入内容或页面标题 | 代码审查 |
| 自定义 HTML 使用沙盒 iframe | 单元测试和代码审查 |
| 外部 URL 主操作经过安全校验 | 单元测试 |
| 不阻断扩展自身页面 | 单元测试和手动验证 |

## 10. 后续方向

后续版本可以考虑：

- 规则组模板库：睡眠、深度工作、备考、周末戒断、社交媒体节食；
- 导入/导出规则组；
- 更细的阻断策略，例如路径级规则或类别模板；
- 更丰富的统计复盘；
- 可选云同步；
- Firefox 支持；
- 在规则模型稳定后评估 `declarativeNetRequest` 动态规则。

这些方向不得改变 MVP 的核心边界：**规则组优先、本地优先、温和但有效、晚安守护只是默认睡眠模板。**
